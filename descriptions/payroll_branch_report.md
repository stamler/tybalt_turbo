# Payroll Branch Report Columns

## STATUS: PLANNED, UNIMPLEMENTED, PENDING REVIEW

### Scope

This spec covers the dynamic branch columns appended to the payroll time report
CSV generated by `/api/reports/payroll_time/{payrollEnding}/{week}`.

It does not change the meaning of the base payroll report columns such as
`hours worked`, `salaryHoursOver44`, `adjustedHoursWorked`, overtime columns, or
time-off columns. Those columns continue to follow the existing payroll report
rules.

### Branch Column Inputs

Each branch column represents regular working time allocated to that branch for
the report week.

Only time entries and committed amendments with time type code `R` or `RT`
contribute to branch columns.

The branch columns must exclude every other time type, including but not limited
to:

- `OP` / PPTO
- `OS` / Sick
- `OV` / Vacation
- `OB` / Bereavement
- `OH` / Stat Holiday
- `OR` / off rotation days
- `RB` / overtime banked
- `OTO` / overtime payout requests

Rows with no branch continue to be grouped under the `Unassigned` branch column.

### Hourly Staff

For hourly staff, branch columns are the actual regular-time hours by branch.

For a given hourly user and report week:

```text
branch value = SUM(hours) where time_types.code IN ('R', 'RT')
```

No cap or normalization is applied to hourly staff branch values.

### Salary Staff

For salary staff, branch columns begin with the same `R` and `RT` branch totals
used for hourly staff.

After those branch totals are calculated, apply a 40-hour cap across all branch
columns for that salary user.

If the salary user's total across branch columns is 40 or less, leave the branch
values unchanged.

If the salary user's total across branch columns exceeds 40:

1. Calculate the excess:

   ```text
   excess = SUM(all branch values for the user) - 40
   ```

2. Subtract the excess from the user's default branch first.

3. If the default branch value would go below zero, set the default branch value
   to zero and carry the remaining excess forward.

4. Subtract any remaining excess from the branch with the highest remaining
   hours.

5. If draining the highest remaining branch still leaves excess, continue
   subtracting from the next highest remaining branch until the user's total
   branch hours equal 40.

6. Branch values must never be negative.

If there is a tie for highest remaining branch, use a deterministic tie-breaker
so the report is stable across runs. Branch name ascending order is preferred.

### Default Branch Handling

The salary cap reduction starts with the user's `admin_profiles.default_branch`.

If the user's default branch has no value in the payroll branch columns for that
week, treat its starting value as zero and immediately apply the remaining
excess to the highest remaining branch values.

If the user has no default branch, skip the default-branch reduction step and
apply the excess to the highest remaining branch values.

### Examples

#### Hourly user

An hourly user has:

```text
Thunder Bay: 32 R hours
Toronto: 12 RT hours
PPTO: 8 hours
```

Branch output:

```text
Thunder Bay: 32
Toronto: 12
```

The PPTO hours do not appear in any branch column.

#### Salary user with excess absorbed by default branch

A salary user whose default branch is Thunder Bay has:

```text
Thunder Bay: 36 R hours
Toronto: 10 R hours
```

The total is 46, so the excess is 6.

Branch output:

```text
Thunder Bay: 30
Toronto: 10
```

#### Salary user where default branch cannot absorb all excess

A salary user whose default branch is Thunder Bay has:

```text
Thunder Bay: 3 R hours
Toronto: 30 R hours
Ottawa: 20 RT hours
```

The total is 53, so the excess is 13. Thunder Bay absorbs 3 and is reduced to
zero. The remaining excess is 10, which is subtracted from Toronto because it
has the highest remaining branch value.

Branch output:

```text
Thunder Bay: 0
Toronto: 20
Ottawa: 20
```

### Test Expectations

Backend tests should cover at least:

- Hourly staff branch columns include only `R` and `RT`.
- Hourly staff branch columns exclude PPTO, Sick, Vacation, Bereavement, and
  Stat Holiday.
- Salary staff branch totals at or below 40 remain unchanged.
- Salary staff branch totals above 40 are reduced from the default branch first.
- Salary staff reduction falls through to the highest remaining branch when the
  default branch cannot absorb the full excess.
- Salary staff reduction is deterministic when two branches tie for the highest
  remaining value.
- Committed amendments follow the same `R` and `RT` inclusion rule as time
  entries.
